ПРИВЕТ!

Этот мини-проект является реализацией условий тестового задания, суть которого заключается в следующем:
 - Приложение на angular 9
 - Карта Винницы (Google maps API).
 - Кроме карты на странице должен быть выпадающий список (html select) объектов ( Кинотеатр Мир, Библиотека Тимирязева, ж/д вокзал, политех, etc)
 - При выборе элемента списка на карте должен появится маркер в точке расположения объекта и карта должна отцентрироваться на этом объекте.
 - Бонусом будет использование ngrx, Observable (rxjs) и написание нескольких тестов.

 Описание реализации по пунктам:
 
 1. В основе используется фреймворк Angular версии 9.1.2.
 
 2. В процессе выполнения задания я столкнулся с трудностью применения Google maps API, а именно: корпорация Google ввела требования по использованию цифрового ключа для 
    предоставления доступа к интерфейсу Google maps API. Обязательным условием для получения API-ключа является выдача корпорации Google данных о своей кредитной карте, что
    на данном этапе является для меня неприемлемым. После некоторого размышления (чтоб не тратить время на создание виртуальных кредитных карт или другие способы получения
    API-ключа "в обход") я решил сосредоточиться на разработке, а вместо Google maps API применить аналогичную библиотеку с таким же функционалом, но без "приколов" с выманиванием
    данных о кредитке. Таким аналогом я выбрал leaflet - на базе этой библитеки весь предписаный заданием функционал интерактивной карты нормально организовался.
    
 3. На странице присутствует выпадающий список, элементы которого собраны в группы по категориям объектов.
 
 4. При выборе элемента списка на карте устанавливается маркер в точку расположения соответствующего объекта и карта центрируется на этом объекте. При перемещениях по локациям
    применяется анимационный эффект "перелёт": т.е. сначала зум карты уменьшается (взлетаем) - центр карты перемещается - зум увеличивается (опускаемся).

 5. "Бонусы" - в виде использования ngrx, Observable (rxjs) - применить хотел... но не успел. В определённые заданием сроки (18 дней на выполнение) мне пришлось интенсивно 
    "накачаться" большим объемом информации. На момент получения задания у меня не было знаний по методам и приёмам работы с Angular и интерактивными картами. Мне пришлось
    на ходу усваивать новое, хватать навыки и тут же приспосабливать их к выполнению условий тестового задания. Поначалу, чтоб избежать усложнения, реализовал всё без применения 
     ngrx и rxjs. Потом нашёл по этим технологиям некоторые туториалы, начал ознакамливаться - но вижу, что в оставшиеся 4 дня до дедлайна сделать вариант приложения с использованием
     ngrx и rxjs уже не успею.
